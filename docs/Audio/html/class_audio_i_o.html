Detailed Description

Audio Input/Output class based on portaudio.

In order to use this class, the following basic initialization is required:

  • create an AudioInterface object
  • create a SceneInterface object
    • only relevant if the SceneInterface is used (and AudioIO is compiled with the define USE_SCENE)
  • create an AudioIO object with reference to the AudioInterface and optionally SceneInterface

To actually use the class for input and output, do the following:

  • Initialize the sound card with the number of in- and output channes you intend to use
    • use init() to enable both input and output
    • use initIn() to only enable input
    • use initOut() to only enable output
    • the init functions can be run multiple times and only the latest is used
  • Add the number of input channels you intend to use to the AudioInterface
    • AudioInterface::addChnCapture()
    • Up to the maximum number of channels in either AudioIO or AudioInterface, audio will be added to the interface
    • The order of channels in the interface is the order in which they were created so, eg. 
      audioIF.addChnCapture("GUID1,first");
audioIF.addChnCapture("GUID2,second");
      will result in the stream from sound card channel 1 to be sent with the ID "GUID1,first" and channel 2 as "GUID2,second"
  • Start the audio capture/playback with start()
    • Note that the AudioInterface channels can be changed while the system is running, the AudioIO channels can only be changed when no audio input/output is taking place.
      It is possible to use stop(), init() and start() with minor skips in audio as a result
    • Input will always be treated as individual channels to transmit
    • Single channel output will mix all received channels
    • Two channel output will render each channel directional (if there is a connection to the Beaming Scene) and mix the resulting binaural signals
