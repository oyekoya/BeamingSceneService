Detailed Description

Interface for asynchronous communication between audio modules.

The AudioInterface class is meant for interfacing different modules by using buffers that work with asynchronous access.
This class should also be used by any program wishing to implement an Audio Client (since it is used by the RakCli class).
The class can be instantiated without any arguments and then used as a reference in the other modules, which can then use and extend it with audio channels channels later.
The class contains 2 sets of separate (but equal) functions:

  • The Capture functions relate to the audio (if any) that is captured by a microphone in order to be transmitted to the server (or generated otherwise for the same purpose, such as in the proxy client). The RakCli class will automatically transmit anything that is in the capture buffers as soon as it is avavilable.
  • The Output functions relate to the audio (if any) that is meant to be output by the client, ie. streams from the server. Once connected to a server, the RakCli class will fill up these buffers with any received audio.

General use

  • Writing to the interface is done by using the functions writeCapture() and writeOutput().
    • If there is not enough room in the buffer, the remaining data will be discarded.
  • Reading from the interface is done by using the functions readCapture() and readOutput() with an indication of how much data to read.
    • If there is not enough available data, only what is available will be read
    • To test how much is available (eg. to avoid dealing with too small bits of data), one can use the functions nAvailCapture() and nAvailOutput().
  • When using these functions one can either use an ID (RakCli uses this method with the ID of each channel) or the channel number (as is done by the AudioIO class looping through channels).
  • If there is an attempt to write to a buffer by ID, and there is no channel with that ID, a channel will be added automatically and a Qt Signal emitted.
  • For looping through channels, the functions nChnCapture() and nChnOutput() will reveal the current number of channels of each type.
